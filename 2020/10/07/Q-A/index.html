发表于 2020-10-07 | 更新于 2020-10-08 | 分类于 else

A1：

首先，你的代码有错，每个数转成二进制都少了最后一位，以下是正确代码：

  #include<stdio.h>
  int main()
  {
      int n,i;
      scanf("%d",&n);
      for(i=8;i>=1;--i)
        printf("%d",(n&(1<<(i-1)))!=0);
      return 0;
  }

以下讨论建立在以上代码上。
首先，这个代码的目的是把十进制数x转化成二进制，具体过程是判断x的二进制上每一位是否为0，为0输出0，否则输出1.

然后我们看具体代码。(1<<(i-1))表示1(在二进制下)右移i位所得的值。这里可以简单理解成 $2^{i-1}$。

&是按位与，将参与运算的两数各对应的二进位相与，只有对应的两个二进位都为1时，结果位才为1。这里就是”取出”x的第i位，如果x的二进制第i位是0，与出来的结果就为零，否则不为0(结果是$2^{i-1}$)。

C语言中“!=”只有两种返回值，0和1，0代表假，1代表真。这里不等号成立表示x的二进制上第i位是1，否则是0。

<<和&都是位运算，更多详情可见《C语言程序设计》P140起及网上各类博客

由上可知，printf("%d",(n&(1<<(i-1)))!=0);这句话就表示判断x的二进制第i位是否为0，不为0输出1，为0输出0，把x的二进制每一位都判断过去就输出了x的二进制表示。注意到计算机在输出的时候从左往右输出，因此for循环中i从7开始，依次递减，先判断、输出最高位。

拓展：
经过尝试，不难发现以上代码仅对小于255(即$2^{8}-1$)的自然数正确,把for(i=8;i>=1;--i)改为for(i=32;i>=1;--i)即可对int范围内的自然数正确。
